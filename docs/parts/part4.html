Data Structures

Duration

10 minutes

Lab goals

The primary goal of this lab is to get an idea of the elegance of discriminated unions in F#.

  1. Analyze a C# project
  2. Convert that project into an F# discriminated union

Steps

Convert a C# project into an F# discriminated union

Our goal is to simplify a C# project into a single discriminated union in F#

  1. Locate the starter project contained in the Exercise 4/Start folder included with this exercise and open Registration.sln.
  2. Open the F# REPL
  3. Analyze the existing C# project. It has four separate classes to manage different types of conference attendees, an abstract base class Registrant and three sub-types: Speakers, Volunteers, and Attendee. Two of the classes have some unique data associated with them.
  4. Your goal is to combine all four of these classes, with their unique data requirements, into a single F# discriminated union named Registrant.

Hint #1

The DU should contain types for the attendees, speakers, and volunteers

Hint #2

The speaker type will include two fields: a room name and number they are speaking in.

Show Code 

type Registrant = 
    | Attendee of email:string
    | Speaker of room:int * title:string
    | Volunteer

Summary

During this lab you have gained a deeper understanding of how discriminated unions help to make code more compact and easy to read.
